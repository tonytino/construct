# Migration: v0.1.0 -> v0.2.0

## Version

- **From:** 0.1.0
- **To:** 0.2.0

## Breaking Changes

- TanStack packages are now pinned to `~1.114.3` via `pnpm.overrides`. Version 1.167+ dropped vinxi support and will break the build. Do not remove or loosen these overrides.
- `DATABASE_URL` in CI is now provided by the `CI_E2E_DATABASE_URL` secret. If your instance has its own CI, rename the secret to match.

## General Rule

Whenever this guide says "copy X from construct," copy the file verbatim rather than hand-writing its contents. Hand-writing drifts; copying does not. After copying, the only permitted edit is merging with existing instance-owned content (e.g. keeping an instance's customization of `AGENTS.md` intact).

## Migration Steps

Follow these steps in order. Run `pnpm preflight` after completing all steps.

### 1. Pin TanStack packages via pnpm overrides

Copy the `pnpm.overrides` block from construct's `package.json` into the instance's `package.json`. This pins 17 `@tanstack/*` packages to `~1.114.3`. If the instance already has a `pnpm.overrides` section, merge the entries without removing existing overrides.

### 2. Add `@tanstack/router-cli` as a dev dependency

Add `@tanstack/router-cli` at `~1.114.3` to `devDependencies`. Do not run `pnpm install` yet — more scripts are added below.

### 3. Pin the package manager

Add `"packageManager": "pnpm@9.15.0"` to `package.json` at the top level. This guarantees the same pnpm version across local dev, CI, and agents.

### 4. Add `tsr.config.json`

Copy `tsr.config.json` from construct into the instance root. It points the route tree generator at `app/routes/` with output to `app/routeTree.gen.ts`.

### 5. Add the `postinstall` and `prepare` scripts

In `package.json`, add to the `scripts` block:

```json
"postinstall": "tsr generate",
"prepare": "lefthook install"
```

`postinstall` regenerates the route tree after install. `prepare` installs pre-commit hooks (see step 7).

### 6. Add the `preflight` script

Copy the `preflight` script verbatim from construct's `package.json`. At v0.2.0 it is:

```json
"preflight": "biome check . && tsc --noEmit && vitest run"
```

Do not hand-write this — copy from construct to avoid drift.

### 7. Add Lefthook pre-commit hooks

1. Add `"lefthook": "^1.11.0"` to `devDependencies`.
2. Copy `lefthook.yml` verbatim from construct's repo root.
3. After `pnpm install` (step 10) the `prepare` script will install the git hooks automatically.

### 8. Update `.gitignore`

Merge the following entries into the instance's `.gitignore` if not already present:

- `node_modules/`
- `.output/`
- `.vinxi/`
- `dist/`
- `.env*`
- `routeTree.gen.ts`
- `app.config.timestamp_*`
- `.claude/`

### 9. Update `biome.json`

Add `app.config.timestamp_*` and `.claude/` to the `files.ignore` array in `biome.json`. Do not remove existing ignore entries.

### 10. Install

Run `pnpm install` (not `--frozen-lockfile` — the lockfile will change). This triggers:

- `postinstall` -> `tsr generate` -> produces `app/routeTree.gen.ts`
- `prepare` -> `lefthook install` -> installs git pre-commit hooks

### 11. Add error boundary and 404 handling to `app/routes/__root.tsx`

Add `notFoundComponent` and `errorComponent` to the root route. Copy the implementations from construct's `app/routes/__root.tsx`. If the instance has customized this file, integrate the components without overwriting existing layout code.

### 12. Add example server function to `app/routes/index.tsx`

If the instance has not customized `app/routes/index.tsx` beyond the scaffold default, copy construct's version (which includes an example server function). If the instance has customized it, skip this step.

### 13. Add example utilities and tests

Copy these files from construct into the instance, but do not overwrite if the instance already has files at these paths:

- `app/utils/format.ts`
- `app/utils/format.test.ts`
- `app/components/Greeting.tsx`
- `app/components/Greeting.test.tsx`

### 14. Update AGENTS.md and docs

Propagate `AGENTS.md` and the following sub-docs from construct:

- `docs/agents/api.md` — expanded API docs with decision table
- `docs/agents/dependencies.md` — new dependency governance doc
- `docs/agents/releases.md` — new release convention doc
- `docs/agents/tooling.md` — new Biome/preflight/Lefthook usage doc
- `docs/agents/tasks.md` — adds `pnpm preflight` and pre-commit hook section
- `docs/agents/propagation.md` — `.construct` version tracking, migration guide precedence
- `docs/agents/environment.md` — if updated

If the instance has customized `AGENTS.md` (e.g. project-specific hard rules), merge construct's Sub-Doc Index and "Template Version Tracking" section into the existing file. Preserve project-specific customizations.

**Do not copy `docs/decisions/` or `docs/migrations/`** — those are template-only and do not propagate to instances.

### 15. Update CI workflow

Copy `.github/workflows/ci.yml` verbatim from construct. Changes: `--frozen-lockfile`, pnpm cache, and E2E conditional on `CI_E2E_DATABASE_URL` secret. If your instance uses a different CI secret name, update it after copying.

### 16. Add validate-template CI workflow

Instances do not need this workflow — it is template-specific (validates construct scaffolding). Skip this step.

### 17. Add PR template

Copy `.github/pull_request_template.md` verbatim from construct.

### 18. Add release-check CI workflow

Instances with their own release conventions can copy `.github/workflows/release-check.yml` if they want automated enforcement that version bumps include a CHANGELOG entry and migration guide. Optional.

### 19. Update `.construct` version

Set `constructVersion` to `"0.2.0"` in the instance's `.construct` file.

### 20. Verify

Run `pnpm preflight` and `pnpm build`. Fix any issues before marking the migration complete.
